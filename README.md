## рҹ§  Arquitectura del proyecto

El proyecto estГЎ organizado siguiendo una arquitectura modular basada en funcionalidades (feature-based structure), lo que permite escalar la aplicaciГіn de forma ordenada y mantenible.

-  `features/`: contiene las distintas secciones de la aplicaciГіn (hero, reviews, locations, etc.), agrupando lГіgica y UI por funcionalidad  
-  `components/`: componentes reutilizables separados por responsabilidad (common y layout)  
-  `theme/`: configuraciГіn centralizada de estilos y theming  
-  `assets/`: recursos estГЎticos como imГЎgenes y logos  
-  `data/`: manejo de datos desacoplado mediante JSON  

Esta organizaciГіn favorece la reutilizaciГіn, escalabilidad y claridad del cГіdigo en proyectos de mayor tamaГұo.
